## 목적

이 문서는 Gemini `File Search`를 사내 서비스나 운영 환경에서 안정적으로 관리하기 위한 짧은 운영 가이드입니다.

---

## 운영 원칙

- 문서는 성격별로 `store`를 분리합니다.
- 검색 품질보다 먼저 문서 분류 기준을 정합니다.
- 모든 문서에는 최소 메타데이터를 부여합니다.
- 답변에는 가능하면 출처를 함께 표시합니다.
- 비용은 저장량보다 검색 시 입력 토큰을 더 주의해서 봅니다.

---

## 권장 store 분리 기준

한 store에 너무 다양한 문서를 섞지 않는 것이 중요합니다.

예:

- `prod-hr-docs`
- `prod-policy-docs`
- `prod-tech-docs`
- `prod-cs-guides`

권장 기준:

- 부서 기준 분리
- 업무 도메인 기준 분리
- 서비스 환경 기준 분리 (`dev`, `stg`, `prod`)

---

## 필수 메타데이터

최소한 아래 항목은 공통으로 관리하는 것을 권장합니다.

- `department`
- `category`
- `owner`
- `version`
- `updated_at`

선택 항목:

- `product`
- `year`
- `language`
- `status`

메타데이터를 붙이면 검색 범위를 좁히고 운영 추적이 쉬워집니다.

---

## 문서 등록 기준

문서 등록 전 확인:

- 최신 문서인지 확인
- 중복 문서인지 확인
- 문서명 규칙이 맞는지 확인
- 적절한 store가 선택되었는지 확인
- 메타데이터가 누락되지 않았는지 확인

문서 등록 후 확인:

- 인덱싱 완료 여부
- 검색 결과 정상 여부
- citation 확인 여부
- 관련 없는 문서 검색 여부

---

## 검색 품질 운영 기준

검색 품질이 낮을 때 먼저 확인할 항목:

- store에 너무 많은 주제의 문서가 섞여 있는가
- 메타데이터 필터 없이 검색하고 있는가
- 문서 청킹이 너무 크거나 작은가
- 질문이 너무 짧거나 모호한가

우선순위 대응:

1. store 분리 재검토
2. 메타데이터 필터 적용
3. 질의문 개선
4. 청킹 설정 조정

---

## 비용 관리 기준

중점 관리 항목:

- 최초 인덱싱 비용
- 검색 시 입력 토큰 비용
- store 총 용량
- 문서 증가 속도

운영 팁:

- 불필요한 문서는 주기적으로 삭제
- 오래된 버전 문서는 교체 정책 적용
- 자주 쓰는 질의는 캐싱 검토
- 너무 큰 store는 분리 검토

---

## 출처 표시 원칙

가능하면 응답에 아래 정보를 함께 노출합니다.

- 문서명
- 문서 버전
- 참조 위치 또는 citation 정보

장점:

- 사용자 신뢰도 향상
- 잘못된 답변 추적 가능
- 운영 이슈 분석 쉬움

---

## 권장 운영 절차

### 신규 문서 등록

1. 문서 검수
2. store 선택
3. 메타데이터 부여
4. 업로드 및 인덱싱
5. 테스트 질의 수행
6. 검색 품질 확인

### 문서 개정

1. 기존 문서 확인
2. 새 버전 업로드
3. 버전 메타데이터 반영
4. 이전 버전 유지 여부 결정
5. 검색 결과 비교 확인

### 문서 삭제

1. 삭제 대상 확인
2. 영향 범위 확인
3. store 내 문서 삭제
4. 검색 결과 재점검

---

## 장애/이슈 대응 체크

### 검색이 부정확한 경우

- store 분리 상태 확인
- 메타데이터 누락 확인
- 청킹 설정 확인
- 테스트 질문 재작성

### 관련 없는 문서가 검색되는 경우

- 문서 분류 재검토
- metadata filter 적용 여부 확인
- 공통 키워드가 많은 문서군 분리

### 비용이 급증하는 경우

- 대용량 문서 유입 여부 확인
- 반복 질의량 확인
- 너무 넓은 검색 범위 여부 확인

---

## 제한 사항

- Live API에서는 지원되지 않음
- 다른 일부 도구와 동시 사용 제한이 있음
- 파일당 최대 크기: `100 MB`
- store 용량은 사용자 티어에 따라 제한됨
- store는 `20 GB 이하` 유지 권장

---

## 운영 결론

File Search 운영의 핵심은 업로드 자체가 아니라 아래 3가지입니다.

- store 분리 기준
- 메타데이터 표준화
- 검색 품질 및 비용 모니터링

이 세 가지를 먼저 잡아두면 File Search 기반 문서 검색 서비스를 훨씬 안정적으로 운영할 수 있습니다.
